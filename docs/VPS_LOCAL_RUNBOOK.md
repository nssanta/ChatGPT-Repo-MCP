# VPS Local Runbook

This VPS uses:

- MCP repo: `/opt/evaai/ChatGPT-Repo-MCP`
- Target repo: `/opt/evaai/Ai_Agent_Base`
- Local MCP endpoint: `http://127.0.0.1:2091/mcp`
- Public quick tunnel: generated by `cloudflared` and visible in service logs

## Start / Restart

```bash
systemctl restart chatrepo-mcp
systemctl restart chatrepo-mcp-tunnel
```

## Status

```bash
systemctl status chatrepo-mcp --no-pager
systemctl status chatrepo-mcp-tunnel --no-pager
```

## Get Current ChatGPT URL

```bash
journalctl -u chatrepo-mcp-tunnel -n 80 --no-pager | grep -o 'https://[^ ]*trycloudflare.com' | tail -1
```

Use the printed URL with `/mcp` appended, for example:

```text
https://example.trycloudflare.com/mcp
```

ChatGPT auth mode: `No authentication`.

## Verify Tools

```bash
cd /opt/evaai/ChatGPT-Repo-MCP
.venv/bin/python scripts/check_tools.py "https://CURRENT.trycloudflare.com/mcp"
```

Expected: 19 tools including `repo_info`, `read_text_file`, `find_files`, `git_blame`, and `git_grep`.

## Logs

```bash
journalctl -u chatrepo-mcp -f
journalctl -u chatrepo-mcp-tunnel -f
```

## Notes

Quick tunnels are free and fast, but the URL changes when `chatrepo-mcp-tunnel` restarts. For a stable URL, replace quick tunnel with a named Cloudflare Tunnel or put a domain/reverse proxy in front.
